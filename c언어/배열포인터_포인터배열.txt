포인터 배열 -> 포인터를 배열의 요소로 가지는 배열.
즉, 포인터 변수를 저장할 수 있는 배열을 의미.
--> 포인터 배열은 주소값들을 저장하는 배열 이다

선언방법 (type)* array[n]
우선 순위 : type [n] > (type)*
array는 배열인데, 포인터(주소값)를 저장한다.

장점 : 문자열등과 같이 큰 길이에 데이터들을 포인터로 가르키게하고 해당 포인터들을 다시 배열로 묶어서 관리하기 쉽게 할때 사용하면 좋다





배열 포인터 -> 배열을 가리키는 하나의 포인터 
---> 배열 포인터는 배열의 시작주소값을 저장할 수 있는 포인터 이다

쓰는이유 : 2차원이상 배열에서 포인터로 배열처럼 인덱싱할려고 

2차원으로 선언한 배열이 있다면 이것을 배열 포인터로 가리키게 하여 접근 -> 결국 2차원이상 배열을 포인터로 가리키는 것

선언방법 (type) (* array)[n]
우선 순위 : (*array) > type [n]
array는 포인터인다. (포인터 -> 주소값을 저장)
따라서 type [n]자료형에 주소값을 담는다.
